# ป้องกัน IT แอบแก้เงินเดือนตัวเอง

1. ไม่อนุญาตให้แก้ไขข้อมูลผ่าน Database โดยตรง

2. กำหนดสิทธิ์ให้ Application Account
เป็นผู้แก้ไขข้อมูลเท่านั้น

3. ผู้ใช้งานต้อง Login และมี Role
เช่น HR หรือ Payroll Manager

4. ทุกการแก้ไขต้องถูกบันทึกใน Audit Trail

5. หากพนักงาน IT ต้องการแก้ไขเงินเดือน
ต้องได้รับการอนุมัติจาก HR Manager
หรือใช้หลักการ Four-Eyes Principle
(ผู้ขอและผู้อนุมัติต้องเป็นคนละคน)

6. เปิด Database Audit Log
เพื่อตรวจสอบการเข้าถึงฐานข้อมูลโดยตรง
