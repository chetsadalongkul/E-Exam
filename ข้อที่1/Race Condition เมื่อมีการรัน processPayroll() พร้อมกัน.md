# จุดผิดพลาดที่ 3 : Race Condition

ปัญหา
หากมีการเรียกใช้ processPayroll() พร้อมกันหลายครั้งสำหรับพนักงานคนเดียวกัน อาจเกิดปัญหาการอัปเดตยอดเงินซ้อนทับกัน (Lost Update)
ส่งผลให้ยอดเงินคงเหลือในฐานข้อมูลไม่ถูกต้อง

วิธีแก้
ใช้ Transaction และ Row Lock (FOR UPDATE) เพื่อป้องกันไม่ให้หลาย Process เข้ามาแก้ไขข้อมูลแถวเดียวกันพร้อมกัน
